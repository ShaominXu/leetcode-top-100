    滑动窗口¶

    滑动窗口（Sliding Window）是一种高效处理序列数据的技术，通过维护一个固定或可变大小的"窗口"在数据中滑动，逐步求解问题。

    核心思想¶

    使用两个指针（快指针、慢指针）维护一个窗口： - 快指针：扩展窗口，引入新元素 - 慢指针：收缩窗口，移除过期元素 - 当窗口内满足条件时，记录结果或更新答案

    主要应用场景¶

    1. 最优子串/子数组问题¶

    • 无重复字符的最长子串
    • 最小覆盖子串
    • 找出字符串中所有字母异位词

    2. 固定窗口大小¶

    • 滑动窗口最大值
    • 串联单词的子串

    3. 可变窗口大小¶

    • 长度最小的子数组
    • 最小覆盖子串

    工作流程¶

    输入: "abcde"，目标: 找无重复字符的最长子串

i=0, j=0: "a"              → 长度1
i=0, j=1: "ab"             → 长度2
i=0, j=2: "abc"            → 长度3
i=0, j=3: "abcd"           → 长度4
i=0, j=4: "abcde"          → 长度5

    关键技巧¶

    • ✨ 哈希表存储：快速查找元素及其位置
    • ✨ 条件判断：什么时候扩展，什么时候收缩
    • ✨ 指针管理：准确移动左右指针
    • ✨ 结果更新：在合适位置更新最优值

    时间复杂度优势¶

    • 朴素方案：O(n²) 或 O(n³)
    • 滑动窗口：O(n) 或 O(n log n)

    每个元素最多被访问两次（一次由快指针，一次由慢指针）

    模板代码¶

    def slidingWindow(s: str) -> int:
    left = 0
    window = {}  # 记录窗口内的字符
    result = 0

    for right in range(len(s)):
        # 扩展窗口
        c = s[right]
        window[c] = window.get(c, 0) + 1

        # 收缩窗口
        while len(window) > k:  # 或其他收缩条件
            c = s[left]
            window[c] -= 1
            if window[c] == 0:
                del window[c]
            left += 1

        # 更新结果
        result = max(result, right - left + 1)

    return result

    常见陷阱¶

    • ❌ 忘记初始化窗口的数据结构
    • ❌ 收缩条件判断错误
    • ❌ 指针位置更新时序不对
    • ❌ 边界条件处理不周

    实战策略¶

    1. 明确窗口维护的条件
    2. 选择合适的数据结构（哈希表、计数数组等）
    3. 确定何时扩展和收缩
    4. 在正确的时刻更新答案
